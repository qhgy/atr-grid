# 项目总结：515880 交易系统（dev）

日期：2026-06-10
仓库：atr-grid（dev 分支 = 代码，main 分支 = 页面与调度）
目标：用代码固定纪律，实现 **胜率 × 赔率（长期期望）最大化**，人工确认执行，绝不自动下单

---

## 一、这个系统是什么

围绕 `SH515880`（AI 硬件 ETF）的一套**证据优先**的交易辅助系统：

```text
任何规则上线前，必须先通过带成本的历史回测；
任何参数改动，必须先过双机制检验，样本外劣化就回退；
每个交易日 11:37 / 15:07 自动生成操作建议，人看完再动手。
```

它与旧的 `atr_grid` 包相互独立，只复用了 `core/market_data.py` 的行情链路（雪球前复权 → akshare 强制前复权 → 本地快照三级降级）。

## 二、策略：三层资金 + 显式状态机

| 层 | 规则 | 依据 |
|---|---|---|
| **底仓（趋势层）** | 收盘连续 5 日站上/跌破 MA200 才确认趋势开关，分批进出；仓位按"当前波动 vs 自身近一年波动中位数"自动缩放 | Faber (2007) 长期均线择时；Moreira-Muir (2017) 波动率管理（改良为相对模式） |
| **机动仓（反转层）** | 上行趋势中高卖一档（收盘 +1.3×ATR），回落接回（卖价 −1.3×ATR）；价格向上脱离则弃轮不追高；接回后续跌超 1×ATR 冻结一切买入，收盘站回 MA20 解冻 | A 股短期反转效应强（Liu-Stambaugh-Yuan 2019） |
| **现金地板** | 买单不得击穿总资产 20%；20 日回撤 ≥10% 时可动用一半（应急通道） | 尾部风险预留 |
| **指数过滤** | 深成指 + 创业板均收于 MA20 下方且 5 日收益为负 → 只许完成接回，禁止新开买仓 | 市场环境 regime filter |

状态机：`IDLE（可开新轮）→ TRIMMED（已卖等接回）→ 接回成交回到 IDLE / 向上脱离弃轮 / 续跌冻结 FROZEN → 站回 MA20 解冻`。总纲里所有"接回是完成上一轮不是抄底"之类的文字纪律，全部变成了可测试的代码。

## 三、关键发现（证据闭环的价值）

1. **状态机卡死 bug**：首版规则下，2022 年一笔接回单（限价 ¥0.435，现价 ¥1.63）因价格上涨脱离永远挂起，机动仓停转 4 年。补上"弃轮"规则后轮次从 14 → 45。**这个 bug 靠人眼盯盘永远发现不了，回测一天就暴露了。**
2. **成分股换血 = 两个资产**（用户洞察，数据证实）：2024 年前后年化波动 30%→42%、对创业板 beta 0.72→0.98、收益偏度翻正。因此旧数据**只用于验生存，不用于预测收益**。
3. **固定波动率目标在"变身"资产上失效**：30% 的绝对阈值撞上 42% 新常态，底仓常年被错杀半仓。改为相对模式后，参数扫描前六名中五个是相对模式——用户的直觉被数据正面证实。
4. **3 日趋势确认太敏感**：参数扫描中 confirm=5 包揽前十，证明 whipsaw（MA200 附近反复打脸）是旧默认的主要亏损源。

## 四、回测证据（双机制协议，36 组参数全检验）

入选参数：相对波动 + grid_k=1.3 + 底仓 60% + 5 日确认（10 万初始资金，含佣金滑点，T+1/整手/一字板约束）：

| 段 | 策略 | 买入持有 | 含义 |
|---|---|---|---|
| 压力段 2020~23（芯片牛熊） | CAGR -4.3%，**回撤 -22.5%** | CAGR -6.4%，回撤 -45.9% | 题材崩塌时活得下来 |
| 训练段 2024+（AI 时代） | CAGR +23.4%，Sharpe 0.95 | CAGR +77.1% | 牛市拿到一部分 |
| 样本外（最近 120 日盲测） | 年化 +128%，回撤 -6.3%，Sharpe 2.75 | 年化 +165%，回撤 -8.2% | 无过拟合劣化 |

**期望管理（必读）**：单边大牛市里这套系统一定跑不赢死拿不动——这是它为"下一次 -46% 崩塌只挨一半"支付的保险费。胜率赔率最大化指长期期望，不是每年都赢。报告里轮次胜率 100% 是构造性的（接回价必然低于卖价），弃轮成本只体现在组合层，**最终裁判永远是 CAGR / MaxDD / Sharpe**。

## 五、工程与质量

- `dev/` 共 18 个模块文件：config（参数单一来源）/ datafeed（复权统一+离线快照）/ indicators / strategy（states+rules+engine 纯函数）/ backtest（broker+runner+metrics+walkforward+report）
- **40 个单元测试**（全仓库 110 个全绿）：状态机全转移路径、撮合边界（T+1/整手/佣金下限/一字板）、规则黄金值、端到端资金守恒
- 决策无前视：t 日收盘决策 → t+1 日撮合；回测可用 `--offline` 离线复现
- 数据降级显式告警（不静默吞错），雪球 cookie 失效自动落 akshare（已强制前复权）

## 六、日常使用

```powershell
uv run python -m dev signal     # 每日信号：状态 + 明日操作单 + 中文原因链
uv run python -m dev backtest   # 全历史带成本回测报告（dev/reports/）
uv run python -m dev scan       # 双机制参数检验（改参数前必跑）
uv run pytest tests/dev -q      # 单元测试
```
