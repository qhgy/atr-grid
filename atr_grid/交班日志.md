## 一、改动清单（what + why）

### 新增

| 文件 | 说明 | 为什么 |
|------|------|--------|
| `config.py` | `GridConfig` dataclass，集中管理全部策略参数 | 原来调一个阈值要翻 3~4 个文件找硬编码，容易漏改；集中后调参只改一处 |

### 修改

| 文件 | 改动 | 为什么 |
|------|------|--------|
| `indicators.py` | `bb_middle = frame["ma20"]` 替代重复 rolling(20)；窗口期从 config 取 | 原来 `close.rolling(20).mean()` 算了两次，浪费且一旦窗口改了容易只改一处 |
| `data.py` | `_to_float` 返回 `math.nan` 而非 `0.0` | **bug 修复**：原来无效 K 线值变成 0.0 参与 ATR/布林带计算，会让指标失真；NaN 让 pandas 自动跳过 |
| `data.py` | `_extract_trade_date` 加 `tz=UTC+8` | 原来用系统本地时区，部署环境时区不是上海时日期会偏一天 |
| `data.py` | `instrument_type`/`price_precision` 从 config 取 | 原来硬编码 `"etf"`/`3`，如果要支持股票（precision=2）就要改源码 |
| `engine.py` | `_PlanContext` dataclass 替代 dict 传递共享字段 | dict 无类型检查、无 IDE 补全，且 warnings 列表共享引用可能被 append 污染 |
| `engine.py` | `_make_plan` 工厂 + 三个分支函数 | 原来 4 处各写 40+ 行 GridPlan 构造，大量重复字段赋值，改一个字段要改 4 处 |
| `engine.py` | replay 预算一次 `build_indicator_frame`，循环内 `iloc[:i+1]` 切片 | 原来每天重建 DataFrame + 重算所有 rolling，O(n×lookback) 冗余计算 |
| `engine.py` | `0.001` → `10**-precision`；`59` → `cfg.ma_long_window - 1` | 消除残留魔法数，让含义自解释 |
| `regime.py` | slope 阈值、lookback 从 config 取 | 同上：调参不翻源码 |
| `report.py` | `_base_csv_row` 提取公共字段；`_pct_change` 返回 `float|None` | CSV 行 3 处重复 15+ 字段；混合 `float|str` 返回类型让下游无法做数值运算 |
| `report.py` | `_fmt_levels` → `fmt_levels`（公开） | cli.py 有一份重复的 `_fmt_ladder`，统一为一个公开函数 |
| `cli.py` | 删 `_fmt_ladder`，改用 `report.fmt_levels` | 消除重复 |
| `__init__.py` | 导出 `GridConfig`、`DEFAULT_CONFIG` | 外部调用方可以传自定义配置 |

## 二、风险披露（行为变更）

| 变更 | 影响范围 | 风险 |
|------|----------|------|
| `_to_float` 返回 NaN 而非 0.0 | 所有经过 `_normalize_rows` 的数据 | 如果上游数据确实有合法的缺失值，现在不会被当作 0 参与计算，指标可能变成 NaN → 进入 disabled 分支。**这是正确行为**，但如果某个 ETF 历史数据有大量缺失值，会比以前更频繁触发 disabled |
| `fmt_levels` 改为公开函数 | `report.py` 的对外接口新增一个符号 | 如果有外部代码 `from atr_grid.report import _fmt_levels` 会 break（不太可能，原来是下划线私有） |
| `_pct_change` 返回 None 而非 `""` | 所有调用 `_pct_change` 的地方 | 已在 report.py 内部用 `_pct_or_blank` 包一层做 CSV 兼容，对外输出无变化 |
| `_disabled_plan` 的 mode 统一为 `"disabled"` | 原来写了一个恒等表达式 `regime.regime if regime.regime == "disabled" else "disabled"` | 行为完全一致，只是代码更清晰 |
