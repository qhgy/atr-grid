## 四、Cookie 到期处理 SOP

> **触发条件**：运行时出现"xq_a_token 还剩 X 天到期"警告，或数据显示"数据：fallback"

### Step 1 — 浏览器导出新 Cookie

1. 打开 Chrome，访问 https://xueqiu.com 确认已登录
2. 安装扩展 **Cookie Editor**（或 EditThisCookie）
3. 导出格式选 **Netscape**（即 `# Netscape HTTP Cookie File` 头的格式）
4. 保存到：`D:\000000znb\1080\x1080x_attachments\xueqiu.com_cookies.txt`

### Step 2 — 本地验证

```powershell
cd "D:\000trae\A股数据\aaa"
$env:XUEQIU_COOKIE_FILE = "D:\000000znb\1080\x1080x_attachments\xueqiu.com_cookies.txt"
uv run python -m atr_grid plan SH515880 --no-save
# 看到"数据：api | 当前价：¥X.XXX"即成功
```

### Step 3 — 更新 GitHub Secret

```powershell
# 从 cookie 文件解析出 header 格式的 cookie 字符串
$cookieText = Get-Content "D:\000000znb\1080\x1080x_attachments\xueqiu.com_cookies.txt" -Raw

# 用项目内置工具解析（输出 "key=val; key=val" 格式）
$cookieHeader = cd "D:\000trae\A股数据\aaa"; uv run python -c "
from core.xueqiu_session import parse_cookie_text
cookie = parse_cookie_text(open(r'D:\000000znb\1080\x1080x_attachments\xueqiu.com_cookies.txt').read())
print(cookie)
"

# 写入 GitHub Secret
Write-Output $cookieHeader | gh secret set XUEQIUTOKEN --repo qhgy/atr-grid
```

### Step 4 — 触发一次 Actions 验证

```powershell
gh workflow run update-dashboard.yml --repo qhgy/atr-grid
# 约 30 秒后检查：
gh run list --repo qhgy/atr-grid --limit 1
```

### 注意事项

- `xq_a_token` 和 `u` 是核心 auth cookie，过期会导致 API 返回 401
- Netscape 格式文件头必须含 `# Netscape HTTP Cookie File`，否则 `parse_cookie_text` 解析失败
- GitHub Secret 更新后无需重启 Actions，下次运行自动生效

## 六、避坑速查

| 症状 | 原因 | 解法 |
|------|------|------|
| pytest 报 `No module named 'atr_grid.xxx'` | `tests/atr_grid/` 下误放 `__init__.py`，与顶级 `atr_grid/` 包冲突 | 删掉 `tests/atr_grid/__init__.py` |
| 改 engine 后金标准变了 | 大概率是 warnings 列表共享引用 | 所有修改外部 list 的地方改用 `list(x)` 本地副本 |
| 小数运算精度漂移 | 用了 `round()` 或纯 float 运算 | 用 `quantize_price`（基于 Decimal），不要绕过 |
| `数据：fallback` 而非 `api` | cookie 过期或未设 env var | 检查 `XUEQIU_COOKIE_FILE` env var，重新导出 cookie |
| push 被 rejected (fetch first) | Actions 自动 commit 比本地新 | `git pull -X ours --no-rebase` 再 push（sync 脚本已自动处理） |
| Server酱不推送 | `SERVERCHAN_KEY` 未设或价格不临近档位 | 用 `--notify-always` 强制推送测试 |
| README.md 显示 `\\\ash`、`\tr_grid/` 等乱码 | PowerShell heredoc 里的反引号被转义；git commit -m 也会吞特殊字符 | 写临时 .py 文件，用 Python 直接写文件，完全绕开 shell 转义 |

## 二、这次留下的坑，下次别再犯

### 1. "意识到的债要一次性修干净"

上次在 `engine.py` 写了：

```python
# Shared base fields for _make_plan
base = dict(
    context=context, snapshot=snapshot, regime=regime, ...
)
```

注释里已经承认这是 "shared base"，就是在向后来人提示"这里应该抽象"。但你选择了最弱的方式 —— 用 `dict`，再靠 `base["context"]` 访问，丢掉了类型检查、丢掉了 IDE 补全、还埋了"warnings 共享引用可能被 append 污染"的地雷。

> **规则**：凡是 `dict(a=…, b=…, c=…)` 在三个以上调用间传递，立刻改成 dataclass。
> 意识到的时候就修，不要写个注释存档然后交班。

### 2. 恒等表达式 = 写完没看第二遍

```python
mode=regime.regime if regime.regime == "disabled" else "disabled"
```

这行恒等于 `mode="disabled"`。写的那一秒大脑在别处。

> **规则**：完稿后自己把 diff 读一遍，5 分钟能拦住这类低级货。
> 具体做法：`git diff` 从头到尾过，每行问自己"这行在做什么，如果删了会怎样"。

### 3. 魔法数和硬编码散装

典型反例：

- `max(sell_price - atr14, 0.001)` —— `0.001` 是什么？实际上是精度 3 对应的最小 tick，应写 `10 ** -precision`
- `start_index = max(59, ...)` —— `59` 是 MA60 要求 60 根历史，应写 `cfg.ma_long_window - 1`

> **规则**：代码里每出现一个数字常量，问三次"这是什么"。
> 如果讲不清它的物理含义 → 命名或进 `config.py`。

### 4. 重复计算在眼皮底下被放过

`build_plan_from_context` 入口已经 `quantize_price(bb_upper)` 了一次，但 `_build_trend_up_plan` 里又量化一次，`_build_range_plan` 里又量化一次。

> **规则**：量化过的值沿上下文传递（dataclass 字段），不要每次用再重算。
> 重复计算除了慢，还容易因为精度不同引入微妙的 bug。

### 5. 「已通过运行验证」≠ 真的验证了

交班日志里写：

```
$ python -m atr_grid plan SH515880 --no-save     ✅
$ python -m atr_grid replay SH515880 --lookback 30  ✅
```

这只是"程序没 crash"，不是验证。

> **验证闭环至少要三件事**：
> 1. **前后金标准对比** —— 改之前跑一次 replay，记下数字；改之后再跑，数字一致（或你能解释差异）
> 2. **纯函数单测** —— `quantize_price` / `_effective_step` / `classify_regime` 这些纯函数写了就该有测试，不然重构时没有安全网
> 3. **关键分支 smoke** —— disabled / trend_up / trend_down / range 四条路径至少各跑一次

上次这三件一件都没做。下次别这样。

### 6. 交班日志的受众是下一个人，不是你自己

日志里的改动表只写了 what，没写 why。例如 "`_to_float` 返回 NaN 而非 0.0" —— 下一个 AI 读到可能想"返回 0 不是更简单吗"，然后直接回退。

> **规则**：每一条改动后面补一句动机。
> 知道 why，才能判断 when it still applies。

此外你还缺：

- **TL;DR（30 秒摘要）** —— 打开文档能立刻抓到重点
- **风险披露** —— 行为变更（如 `_to_float` 0→NaN、`fmt_levels` 改公开）会影响哪些下游？列清楚
- **待办量化** —— "replay 性能较慢"要给出 "lookback=120 时 X 秒，目标 < Y 秒"，不是靠感觉
